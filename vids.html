1. Analisis Parameter Keparahan & Visual Spoofing (UI Masking)

Konteks keparahan mengalami eskalasi signifikan ketika eksekusi logika intent:// digabungkan dengan manipulasi status antarmuka grafis peramban (UI/UX). Vektor ini mengeksploitasi penanganan elemen transisi layar penuh untuk menyembunyikan komponen navigasi kritis.

Faktor-faktor amplifikasi risiko meliputi:

  • Penyembunyian Address Bar & Notifikasi: Melalui pemanggilan requestFullscreen() yang dipicu secara simultan dengan pengalihan skema, peramban dipaksa merender dokumen pada mode layar penuh terisolasi. Hal ini menghilangkan *Omnibox* (bilah alamat URL) dan menyembunyikan status bar/notifikasi bawaan sistem Android.
  • Reduksi Deteksi Korban (Visual Obfuscation): Ketiadaan indikator konteks asal (*origin indicator*) menyebabkan pengguna tidak dapat memverifikasi keaslian sesi dokumen aktif, meningkatkan efektivitas serangan phishing atau overlay berbahaya secara mutlak.
  • Kombinasi Token Gestur: Interaksi tunggal (klik) yang diberikan pengguna dimanfaatkan secara asinkron untuk memenuhi dua prasyarat keamanan sekaligus: memberikan izin transisi Fullscreen API dan meloloskan parsing pengecekan *user gesture* pada mesin pengalihan Intent.

2. Alur Rantai Kerentanan (Exploit Pipeline)

Akar masalah terjadi melalui tahapan kronologis berikut:

  1. Pengguna berinteraksi dengan elemen pemicu di halaman root yang dikontrol oleh penguji.
  2. Handler menangkap token interaksi dan segera mengeksekusi permintaan Element.requestFullscreen() untuk mengunci tampilan viewport global dan mengeliminasi elemen dekorasi UI peramban.
  3. Secara sinkron, sub-proses menginisiasi panggilan window.open() memanfaatkan pseudo-skema javascript: yang mengarah pada manipulasi properti lokasi internal (intent://).
  4. Blink Engine gagal menegakkan batas isolasi origin pada konteks layar penuh, membiarkan muatan berjalan di atas origin target tanpa hambatan peringatan sistem.

3. Struktur Payload Sintaksis Kombinasi

// Contoh Konstruksi Logika Pengujian Kombinasi API
function triggerChainedAudit() {
  document.documentElement.requestFullscreen().then(() => {
    window.open("javascript:location='intent://google.com#Intent;scheme=https;package=com.sec.android.app.sbrowser;end'");
  });
}

Komponen Analisis Struktur:

  • requestFullscreen() : Menghilangkan konteks visual navigasi eksternal, memaksa area tampilan menjadi eksklusif untuk dokumen yang dieksekusi.
  • javascript:location=intent:// : Memaksa pemrosesan bypass Same-Origin Policy (SOP) di dalam ruang isolasi yang telah disamarkan.

5. Mekanisme Rekayasa Perbaikan (Remediation & Patching)

  • Fullscreen Exit on Navigation: Peramban modern menerapkan aturan ketat di mana mode layar penuh wajib keluar (exit) secara otomatis apabila terjadi peristiwa navigasi lintas domain atau inisiasi skema eksternal (seperti intent://).
  • Persistent Security Indicators: Penambahan mekanisme proteksi UI non-interaktif yang secara periodik memunculkan kembali bar alamat jika terdapat deteksi aktivitas pemanggilan sub-proses IPC atau pemuatan skrip asinkron.
