Chrome Android: Intent Navigation Policy Auditor

Tujuan: Membuktikan kegagalan isolasi User Activation Token dan pelarian tata kelola sandbox pada sub-konteks dokumen.

Skenario 1: Direct User Activation Valid Gesture

Menembakkan intent secara sinkronus tepat saat interaksi klik terjadi. Perilaku Chrome yang diharapkan: ALLOWED (Sesuai spesifikasi).

Skenario 2: Async Task / Thread Break No Gesture

Mengeksekusi intent di dalam microtask (Promise chain) pasca penundaan waktu eksekusi. Menguji apakah transient user activation bocor melalui tumpukan microtask.

Skenario 3: Strict Sandbox Subframe No Privilege

Memaksa pemanggilan intent dari dalam iframe terisolasi tanpa token navigasi top-level. Jika intent terbuka, ini adalah Sandbox Escape (High Severity).

Skenario 4: Double Redirect Race No Gesture

Mengalihkan status navigasi window saat ini ke skema internal data URI, yang kemudian melempar balik payload intent tanpa interaksi sekunder.
