🔬 LAPORAN RISET: CHROMIUM VULNERABILITY

Studi Kasus: CVE / Chromium Tracker Issue 40064598

Komponen Vulnerable Blink > SecurityFeature > ContentSecurityPolicy / Mobile > Intents
Ditemukan Pada Versi Chromium Android v112 (Status: Fixed)
Severity / VRP Reward S2 (Medium-High) / $3,000 VRP Reward
Mekanisme Serangan UXSS via Intent Redirection Bypass

1. Deskripsi Teoritis & Analisis Bug

Celah ini terjadi karena kelemahan validasi pada mekanisme penanganan Android Intent URL Scheme di Chromium Mobile. Penyerang mengeksploitasi fungsi window.open() yang digabungkan dengan skema intent:// dan disisipi instruksi pseudo-skema javascript:location=....

Dampaknya, browser gagal memvalidasi batasan origin (Same-Origin Policy) dan mengeksekusi script eksternal dalam konteks browser target (dalam hal ini mencoba memanggil package Samsung Internet Browser com.sec.android.app.sbrowser).

2. Analisis Kode Sumber (PoC)

Logika manipulasi URL yang digunakan dalam riset ini adalah:

window.open("javascript:location='intent://google.com#Intent;scheme=https;package=com.sec.android.app.sbrowser;end'");

Pada versi Chromium yang rentan (v112 kebawah), rantai eksekusi ini memaksa browser mengeksekusi sub-proses Intent secara tidak aman (Bypass CSP).

4. Solusi & Mitigasi (Patching)

  • Sisi Browser: Tim Google Chromium telah memperketat sanitasi skema javascript: ketika dipanggil bersamaan dengan parsing Android Intent URL di level kode native C++ (Blink Engine).
  • Sisi Developer Web: Selalu terapkan header Content-Security-Policy (CSP) yang ketat, khususnya direktif script-src dan mencegah penggunaan skema URL eksternal yang tidak terdaftar.
